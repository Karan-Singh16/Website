Problem

Satellites output continuous telemetry (temps, voltages, currents, attitude‚Ä¶), often noisy and drifting with operational modes. We need to flag emerging faults early, without spamming operators with false alarms.

Approach

  • Generate realistic, labelled streams of telemetry (normal vs injected faults).
  • Engineer features (rolling stats, deltas, z-scores) per channel.
  • Train an unsupervised anomaly detector with thresholds tuned per subsystem.
  • Stream scored events to a web dashboard with live charts and alert trays.

Architecture

Flow: Simulator ‚Üí Feature Worker ‚Üí Anomaly Scorer ‚Üí Event Bus ‚Üí Web API ‚Üí WebSockets ‚Üí UI

Components are decoupled, so I can swap the detector or bus without touching the UI.

Anomaly Detection

I used Isolation Forest for its speed and robustness to outliers, tuned with a small contamination rate and per-channel thresholds. Rolling windows stabilise the decision function.

// Pseudocode: isolation forest scoring (Python)
X = build_features(telemetry, windows=[30,60,120])   # means, std, diffs, z-scores
model = IsolationForest(n_estimators=200, contamination=0.02, random_state=42).fit(X_ref)
scores = -model.score_samples(X)                     # higher = weirder
alerts = X[scores > per_channel_threshold]

Frontend

  • React UI with a compact grid layout: status cards, sparkline trends, and an alert drawer.
  • WebSocket push keeps the UI under ~100ms behind the event stream on my machine.
  • Keyboard shortcuts to jump through recent alerts and filter by subsystem.

Results

  • Recall: 92‚Äì95% on synthetic faults (thermal runaway, voltage droop, attitude jitter).
  • Latency: ~98ms median sim ‚Üí UI (local), <200ms p95.
  • Noise handling: Rolling features reduced false positives by ~37% vs raw signals.
